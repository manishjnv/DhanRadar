Signal — Architecture Approach

How should the Signal feature be structured in DhanRadar's navigation?

A

Single Scrollable Page /signal

One route. All sections stacked vertically with anchor links — Today's Signal → My Rules → Dip Fund → Journal → Analytics.

Pros

  • Simplest routing
  • Mobile scroll-native
  • One sidebar entry

Cons

  • Very long page
  • Hard to jump between Journal and Today
  • No deep links
B

Tabbed Page /signal?tab=rules Recommended

One sidebar entry. Tabs within the page: Today · Rules · Dip Fund · Journal · Analytics. Tab persisted in URL query param.

Signal page preview
Today Rules Dip Fund Journal Analytics
Market signals · Portfolio context · Rule check · Learning content

Pros

  • Matches existing DhanRadar patterns
  • Each section has full space
  • URL-shareable per tab
  • Clean single sidebar item

Cons

  • Tab count (5) is on the higher side
C

Sub-routes /signal/today, /signal/rules…

Signal becomes a mini-section in the sidebar with 5 nested nav items — Today, Rules, Dip Fund, Journal, Analytics.

Pros

  • Fully bookmarkable URLs
  • Each page independently routable
  • Maximum room per section

Cons

  • Expands sidebar significantly (already 6 items)
  • More routing boilerplate
  • Feels like a separate app, not a feature
