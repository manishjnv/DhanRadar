# Components (build rules)

- One component per concept; **variant properties** (size, variant, state, theme) — not duplicate components.
- **Auto-layout** everywhere; nothing hand-positioned.
- States as variants: default/hover/focus/active/disabled/loading/error/success.
- The **four data-states** (loading/empty/error/success) are first-class variants on every data surface.
- Boolean + instance-swap props for icons/slots. Token-bound fills/text.
- Inventory: Core (button, input, card, table, badge, chip, tabs), DataViz (ScoreRing, AreaChart, Sparkline, Donut, FactorBars, gauges), Patterns (alert, toast, notification, watchlist row, recommendation card, empty/loading/error), AI (search, assistant, explainability, confidence, bull/bear, sources, transparency).
